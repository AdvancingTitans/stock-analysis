# Output Discipline

- 除单专家 lens 或 adversarial 对抗辩论外，市场复盘报告固定顺序：执行摘要 → 大盘指数概览 → 持仓分析（有持仓时）→ 六模块深度复盘（M1-M6）→ 综合持仓建议与风险提示 / 通用市场建议与风险提示。
- 用户明确提出想用哪位投资专家的风格生成报告时，必须完全以相关专家的视角输出报告；不得只在结尾追加专家点评。
- 单专家视角不输出多专家委员会小节，不输出交易计划草案、风险管理意见、组合经理最终意见等委员会内容。
- 专家视角可识别专家名称、英文名、中文名、别名或框架 id；不得模仿身份声明或虚构专家发言。
- trading 入口按“用户完整持仓输入 → 本技能投资记忆 → 无持仓”的优先级决定是否输出持仓绩效内容；用户完整输入必须覆盖写入 `~/.stock_analysis/profile.json` 或 `STOCK_ANALYSIS_PROFILE`。
- 用户主动提供持仓但信息不完整时，等待用户交互输入并只提问一次；用户未提供持仓且没有完整投资记忆时，只输出普通市场复盘报告。
- 只有股票代码、买入日期、买入数量或买入金额三项完整时，才输出持仓绩效内容；缺失任意一项，不得进行收益计算。
- 用户主动提供持仓信息但不完整时，才进入确认流程；确认流程只提问一次，精准指出缺失项。
- 用户补齐或修改后的持仓信息完整时，必须保存到本地投资记忆，并告知“投资记忆已保存本地。下次将默认按这份投资记忆分析持仓；如需清空投资记忆请反馈。”
- 如果用户新提供的信息与之前保存的投资记忆不一致，必须先确认信息完整性；确认信息完整性后，优先以用户新提供的信息为准，覆盖写入投资记忆。不完整的新信息不得覆盖已有完整投资记忆。
- 所有行情、汇率、资金流、板块、新闻都必须在 evidence JSON 中记录来源与交易日。
- 价格、涨跌额、涨跌幅缺失时禁止显示 `0.00`；正式报告对应单元格留空。
- fallback、浏览器降级、限流重试写入 evidence/diagnose，不进入正式研报正文。
- 早盘、盘中、午间、盘后报告正文都不得输出“证据附录”章节；需要审计时只查看 `--emit-evidence` 生成的 JSON 文件。
- summary、key-points、full 都必须保留完整免责声明。
- `quality_score < 60` 的 full 复盘仍保留固定章节顺序；缺失模块在对应 M1-M6 章节内标注“证据暂缺”，不得直接跳过大盘或六模块。
- 正式研报不展示接口来源、API 名称和 fallback 技术轨迹；这些信息仅保留在 evidence/diagnose。
- 指数、持仓、连板梯队优先使用 Markdown 表格。
- 持仓分析由完整投资记忆或完整确认信息触发时，必须输出 `references/template/portfolio-template.md` 定义的三个表格；单个字段缺失时该单元格留空，不得拆成段落描述。
- 当前日期的直接股票持仓可追加公开信息脉冲表；历史日期不得混入当前新闻。
- 缺失字段使用空单元格，不显示 `--`、`未知` 或 `数据不足`。
- 综合持仓建议只在持仓分析成立时输出，必须引用实际持仓、当日浮动盈亏、相对基准表现和集中度；无持仓时输出通用市场建议与风险提示。
- 综合持仓建议与通用市场建议固定包含“现状总结、基准跑赢/跑输、条件化仓位动作、下一交易日观察清单、风险提示”五个小节。
- 条件化仓位动作必须使用条件化表达，下一交易日观察清单必须包含可验证指标。
- 无完整投资记忆或本次持仓信息不完整时给出普通市场复盘报告；若进入确认流程，可在末尾只提问一次，提示补齐股票代码、买入日期、买入数量或买入金额，并在金额或数字含义不清时确认币种是人民币、港币还是美元。
