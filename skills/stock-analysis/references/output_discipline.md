# Output Discipline

- 普通市场复盘报告固定顺序：大盘指数概览 → 6 模块深度复盘 → 通用市场建议与风险提示。
- 持仓分析必须由用户主动触发；没有主动提供持仓信息的用户不会被打扰。
- 只有股票代码、买入日期、买入数量或买入金额三项完整时，才输出持仓绩效内容；缺失任意一项，不得进行收益计算。
- 信息不完整时输出普通市场复盘报告，不包含任何持仓绩效内容，并且只提问一次，精准指出缺失项。
- 所有行情、汇率、资金流、板块、新闻都必须在 evidence JSON 中记录来源与交易日。
- 价格、涨跌额、涨跌幅缺失时禁止显示 `0.00`；正式报告对应单元格留空。
- fallback、浏览器降级、限流重试写入 evidence/diagnose，不进入正式研报正文。
- summary、key-points、full 都必须保留完整免责声明。
- `quality_score < 60` 时仅允许输出简化版报告。
- 正式研报不展示接口来源、API 名称和 fallback 技术轨迹；这些信息仅保留在 evidence/diagnose。
- 指数、持仓、连板梯队优先使用 Markdown 表格。
- 持仓分析被用户主动触发且信息完整时，必须输出 `references/template/portfolio-template.md` 定义的三个表格；单个字段缺失时该单元格留空，不得拆成段落描述。
- 当前日期的直接股票持仓可追加公开信息脉冲表；历史日期不得混入当前新闻或社区数据。
- 社区情绪必须精确匹配标的并过滤低质量内容，有效样本少于 3 条时只写“证据不足”。
- 缺失字段使用空单元格，不显示 `--`、`未知` 或 `数据不足`。
- 综合持仓建议只在持仓分析成立时输出，必须引用实际持仓、当日浮动盈亏、相对基准表现和集中度。
- 综合持仓建议固定包含“基准跑赢/跑输、仓位动作建议、观察清单”三个小节。
- 仓位动作建议必须使用条件化表达，观察清单必须包含下一交易日可验证指标。
- 无持仓或持仓信息不完整时给出普通市场复盘报告；若本轮用户主动提到持仓，可在末尾只提问一次，提示补齐股票代码、买入日期、买入数量或买入金额，并在金额或数字含义不清时确认币种是人民币、港币还是美元。
