# Output Discipline

- 报告固定顺序：大盘指数概览 → 持仓分析 → 6 模块深度复盘 → 综合持仓建议与风险提示。
- 所有行情、汇率、资金流、板块、新闻都必须在 evidence JSON 中记录来源与交易日。
- 价格、涨跌额、涨跌幅缺失时禁止显示 `0.00`；正式报告对应单元格留空。
- fallback、浏览器降级、限流重试写入 evidence/diagnose，不进入正式研报正文。
- summary、key-points、full 都必须保留完整免责声明。
- `quality_score < 60` 时仅允许输出简化版报告。
- 正式研报不展示接口来源、API 名称和 fallback 技术轨迹；这些信息仅保留在 evidence/diagnose。
- 指数、持仓、连板梯队优先使用 Markdown 表格。
- 持仓分析必须输出 `references/template/portfolio-template.md` 定义的三个表格；单个字段缺失时该单元格留空，不得拆成段落描述。
- 缺失字段使用空单元格，不显示 `--`、`未知` 或 `数据不足`。
- 综合持仓建议必须引用实际持仓、当日浮动盈亏、相对基准表现和集中度。
- 综合持仓建议固定包含“基准跑赢/跑输、仓位动作建议、观察清单”三个小节。
- 仓位动作建议必须使用条件化表达，观察清单必须包含下一交易日可验证指标。
- 无持仓时给出通用市场建议，并提示用户提供代码、数量、买入日期或使用 young profile。
