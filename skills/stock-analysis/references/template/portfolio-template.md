# 持仓分析模板

投资记忆优先。默认读取 `~/.young_stock/profile.json` 或 `YOUNG_STOCK_PROFILE`；只有完整投资记忆，或用户补齐并确认的股票代码、买入日期、买入数量或买入金额三项完整时，才按以下顺序原样套用三个 Markdown 表格。单个字段缺失时保留空单元格，不得改写成段落。

如果用户明确提出想用哪位投资专家的风格生成报告，必须完全以相关专家的视角输出报告：表格之后的持仓建议、风险提示和观察清单都服从该专家框架，不得只在结尾追加专家点评。专家视角可识别专家名称、英文名、中文名、别名或 lens id；单专家视角不触发 M7，不输出交易计划草案、风险管理意见、组合经理最终意见等委员会小节；不得模仿身份声明或虚构专家发言。最后章节标题写成 `## {专家中文名}持仓建议与风险提示`。

如果没有完整投资记忆，且用户没有主动提供持仓信息，输出普通市场复盘报告，不套用本模板，不进行收益计算。投资记忆有但信息不完整，或用户主动提供但缺少任意核心项时，只提问一次，精准指出缺失项；若用户只给出数字，必须确认这是买入数量还是买入金额，并确认币种是人民币、港币还是美元。确认后的完整信息必须保存到本地投资记忆，并告知用户“投资记忆已保存本地。下次将默认按这份投资记忆分析持仓；如需清空投资记忆请反馈。”

如果用户新提供的信息与之前保存的投资记忆不一致，先按完整性口径确认信息完整性；确认信息完整性后，优先以用户新提供的信息为准，覆盖写入投资记忆。不完整的新信息不得覆盖已有完整投资记忆。

规则：

- 三个表格仅在投资记忆优先命中或完整持仓信息成立时输出。
- 完整性口径固定为：股票代码、买入日期、买入数量或买入金额。
- 缺失任意一项，不得进行收益计算、浮盈亏计算、年化表现或组合绩效分析。
- 日期没有年份时默认使用当前年份；完整年份必须严格按用户提供值。
- 现价保留原始币种；总市值和总浮盈/亏统一折算为 CNY。
- 跑赢为正数、跑输为负数，单位为百分点（pp）。
- 趋势使用已有 MA5/MA10/MA20 结论，不得根据缺失数据自行推断。
