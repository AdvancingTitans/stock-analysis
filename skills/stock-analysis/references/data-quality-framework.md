# 数据质量框架

## 行情清洗

- `price/open/high/low <= 0`：置为 `None`，增加 `price_anomaly`。
- 个股成交量 `<= 0`：增加 `volume_zero`。
- 个股成交量 `< 1,000`、指数成交量 `< 1,000,000`：保留数值并增加 `volume_anomaly`。
- 指数没有成交量但有成交额：允许作为完整活动字段。
- 空字符串、`-`、`--`：统一转换为 `None`。

## 指数有效性

指数展示前同时检查：

1. 点位必须大于零。
2. 涨跌额与涨跌幅不能同时为空或为零。
3. 数据交易日应与 `_meta.trade_date` 一致。

失败时按市场路由切换数据源；全部失败则隐藏该指数行，并记录 `source_events`。

## Evidence 质量

| 模块 | 分值 | 最低证据 |
|---|---:|---|
| M1 | 20 | 主要指数；涨跌比缺失时必须记录 |
| M2 | 20 | 行业/概念板块或资金数据 |
| M3 | 20 | 涨停/强势候选及成交额证据 |
| M4 | 15 | 跌停、大跌或炸板风险 |
| M5 | 15 | 特征分组或持仓风格 |
| M6 | 10 | 可解释的抗跌样本 |

- `>=80`：完整报告。
- `60-79`：完整报告并标注缺失模块。
- `<60`：仅指数、持仓和风险提示。

## 审计字段

`_meta` 应包含：

- `trade_date`
- `session`
- `quality_score`
- `missing_modules`
- `source_events`
- `style`
- `style_filter`
